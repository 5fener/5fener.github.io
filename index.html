顺序表和链表的比较

发表于 2023-02-21 分类于 数据结构
本文字数： 89 阅读时长 ≈ 1 分钟

顺序表（顺序存储)

  • 优点：支持随机存取、存储密度高

  • 缺点：大片连续空间分配不方便，改变容量不方便

链表（链式存储）

  • 优点：离散的小空间分配方便，改变容量方便
  • 缺点：不可随机存取，存储密度低

静态链表

发表于 2023-02-21 分类于 数据结构
本文字数： 386 阅读时长 ≈ 1 分钟

静态链表：分配一整片连续的内存空间，各个结点集中安置

优点：增删操作不需要大量移动元素

缺点：不能随机存取，只能从头结点开始依次往后查找；容量固定不可变

  • 查找：从头结点出发挨个往后遍历结点

  • 插入位序为 i 的结点：

    • 找到一个空的结点，存入数据元素
    • 从头结点出发找到位序为 i-1 的结点
    • 修改新结点的 next
    • 修改 i-1 号结点的 next

  • 删除某个结点

    • 从头结点出发找到前驱结点
    • 修改前驱结点的游标
    • 被删除结点next设为-2（用一个特殊的数值标记空闲结点）

Linux云服务器远程登录失败

发表于 2023-02-21 分类于 疑难杂症
本文字数： 856 阅读时长 ≈ 1 分钟

在通过 Linux ssh 远程登录华为云服务器时，出现报错WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!

问题分析

  • 这是SSH 使用过程中最常见的错误“警告：远程主机标识已更改”

  • 可能原因

    • 服务器误报：known_hosts当服务器的唯一指纹与首次连接时存储在文件中的指纹不匹配时，通常会出现警告：远程主机识别错误。此密钥更改的一个常见原因是因为更改了 root 密码或重建了 VPS 服务器
    • 更可怕的猜测是中间人攻击，攻击者将使用持有相同 IP 的虚假 SSH 服务器设置在你的连接和服务器之间。然后他们会将你的流量转发到真正的 SSH 服务器，让你不知道他们正在监视和收集您所做的一切

谨慎排除

如果你近期做出了如下改变，那么就是服务器误报而非中间人攻击引起的警告

  • 更改了你的 VPS 密码或 SSH 密钥

  • 重建你的服务器

  • 为多个远程系统分配相同的 IP 地址

  • 销毁了你的服务器，然后启动了一台恰好具有相同 IP 地址的新服务器

  • 切换操作系统或重新安装

解决方案

在排除自己是中间人攻击的受害者后，只需删除文件中的现有条目即可再次连接到你的服务器known_hosts，有几种方法可以做到

  • 使用文本编辑器

  • 你可以使用任何文本编辑器浏览到您的known_hosts位置并删除与你的服务器 IP 对应的行。该known_hosts文件可能存储在以下位置：

    • Windows： %USERPROFILE%\\.ssh or %USERPROFILE%\\ssh
    • Linux 和 macOS： .ssh/known_hosts

  • 在命令行中使用 ssh-keygen

发表于 2023-02-19 分类于 数据结构
本文字数： 673 阅读时长 ≈ 1 分钟

按位查找，获取表 L 中第 i 个位置的元素的值

按值查找，在表 L 中查找具有给定关键字值的元素

时间复杂度

  • 最好情况：目标元素在表头
    • 循环1次，最好时间复杂度 = O(1)
  • 最坏情况：目标元素在表尾
    • 循环n次，最坏时间复杂度 = O(n)
  • 平均情况：假设目标元素出现在任何一个位置的概率相同，都是1/n
    • 平均循环次数 = (n + 1)/2
    • 平均时间复杂度 = O(n)
